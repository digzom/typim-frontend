# ADR-005: Tokenized Style Artifacts and Width Safety

## Status

Accepted

## Context

Reviewer findings identified style artifact gaps and hardcoded layout dimensions that could drift from tokens and trigger mobile regressions.

## Decision

Create style split artifacts in `src/styles/editor/`, `src/styles/preview/`, and `src/styles/ui/`; import them from `src/styles/main.css`. Replace critical hardcoded dimensions with layout token variables in `src/styles/tokens/layout.css`.

## Consequences

### Positive

- Token usage is auditable and safer for responsive changes.
- Style ownership is clearer by area (editor/preview/ui).

### Negative

- More files must stay synchronized during future CSS edits.

### Neutral

- Visual design language remains consistent with existing theme tokens.
